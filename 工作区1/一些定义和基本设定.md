

# LauncherDir 

我们现在暂时使用 bat 启动 Codex。
bat所在的 文件夹会 安置 .cool-system 文件夹 里面存放 系统设置
我们把 bat  所在的 文件夹 叫做 LauncherDir

我们现在的启动器 暂时先放在 C:\Arsenal 文件夹 也就是说 
目前的 LauncherDir 为  C:\Arsenal


`.cool-system` 中存放系统级配置、系统级命令策略、系统级视野策略等全局安全设置。

# 文件结构

我们先临时决定，就是启动的时候使用 bat 启动 Codex
bat 所在的文件夹叫做 LauncherDir
LauncherDir 中存在  .cool-system 文件夹 里面存放 系统级别的设置。

 .cool-system  大致结构：

```
LauncherDir\.cool-system     //系统配置文件夹
│
├─ config.toml
├─ scope.toml
├─ command.toml
│
├─ logs
└─ state
```


### CoolSystemDir
指 LauncherDir\.cool-system

而我们 CMD 呼叫 Codex的文件夹 就叫做 SessionRoot
SessionRoot 文件夹里  的.cool文件夹  面放的是  
Session 级别 设定

.cool文件夹 大致结构

```
SessionRoot\.cool    //SessionRoot 级别
│
├─ config.toml       普通项目配置
├─ scope.toml        文件/文件夹视野策略
├─ command.toml      命令风险策略
│
├─ cache
├─ logs
└─ state
```


.cool-system  和 .cool 
对  Codex 模型不可见的 无法对其操作。 位于 CTool 直接写死。

也就是说 在 SafeMode 开启的时候 ，使用我们的Cool 开头的 permission profile 
Codex 是绝对无法看到 .cool-system  和 .cool

因为 Codex 的阅读工具 是 都是CTool 而 CTool 工具上直接写死。


```
1. 捕获 SessionRoot
2. 找  LauncherDir\.cool-system
3. 找  SessionRoot\.cool
4. 读取 .cool-system/config.toml
5. 读取 .cool/config.toml，确定 CoolWorkspace
6. 读取 .cool-system/scope.toml + .cool/scope.toml
7. 读取 .cool-system/command.toml + .cool/command.toml
8. 构造最终 CToolScope 和 CommandPolicy
```


#
CoolSystemDir = LauncherDir\.cool-system 和 CoolDir = SessionRoot\.cool

# CoolWorkspace

SessionRoot 中 .cool 的 config.toml     里设置

如果没有设置默认等于 SessionRoot


注意 CoolWorkspace 一个重要的功能是  
CToolCommandRequest  工具
都是基于 CoolWorkspace 这个路径来运行的 而不是基于 SessionRoot


这样设计的好处是   同一个工作区域 可以使用不同设定的 Codex来进行不同风格的操作。
