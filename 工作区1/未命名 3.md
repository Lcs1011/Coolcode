MD文档注释
按照行号操作 所以需要 
ctool_diff_map 
配合 
这是它给的理由
  目前只有 ctool_preview_diff，它是很基础的：

  - 输入替换/插入操作
  - 输出简单 diff
  - 不写文件

  但它不是“地图工具”，因为它不擅长回答这些问题：

  - 这个文件的行号结构是什么？
  - 我要注释第 37 行，附近上下文是什么？
  - 插入后行号会怎么偏移？
  - 多个注释会不会互相影响？
  - 有没有插进代码块、表格、已有 mark 注释里面？
  - 同一段文本重复出现时，应该选哪个位置？
  - 修改前后，哪些行对应哪些行？

  所以如果要做 MD 注释工具，我建议先做一个更强的只读 diff/map 工具，比如：

  ctool_diff_map

  它不写文件，只生成“修改地图”。

  它可以输入：

  {
    "path": "xxx.md",
    "operations": [
      {
        "operation": "insert_after_line",
        "line_number": 37,
        "content": "<mark style=\"background:#d3f8b6\">这里需要补定义</mark>"
      }
    ],
    "context_lines": 3
  }

  它输出：

  - 原始文件总行数
  - 修改后总行数
  - 每个操作是否合法
  - 每个操作前后的上下文
  - 原始行号 -> 新行号映射
  - 新行号 -> 原始行号映射
  - diff 预览
  - 风险提示，比如：
      - 插入到代码块内
      - 插入到表格中
      - 附近已有 <mark>
      - 同一行已有注释
      - 多个操作落在同一行
      - 操作顺序会造成行号偏移

  然后注释工具就可以复用它：

  1. ctool_diff_map 先看修改地图
  2. ctool_apply_annotation 再实际写注释

  我建议它属于 基础功能增强类：只读型 CTool，因为它只读文件、模拟修改、不落盘。
  真正写入注释另做一个写类工具。
